# 🤖 Telegram Verification Bot

Полнофункциональный бот для верификации пользователей перед регистрацией на сайте transformation-map.com с максимальной безопасностью и UX.

## ✨ Функционал

### 🔐 Безопасность
- **AES-256 шифрование** всех передаваемых данных
- **Проверка возраста аккаунта** (минимум 7 дней)
- **Rate limiting** (5 запросов в минуту на пользователя)
- **Защита от повторной верификации** по telegram_id
- **Лимит попыток** (3 попытки в час)
- **Математическая капча** для защиты от ботов
- **🔐 (02.08.2025) Система повторных попыток капчи** (3 попытки → блок 5 мин)
- **Время жизни токена** (30 минут)

### 📊 Аналитика и Мониторинг
- **Детальная статистика** конверсии по этапам
- **Отслеживание времени** прохождения процесса
- **Логирование всех событий** для анализа
- **Health check endpoint** для мониторинга
- **Метрики неудач** с причинами

### 🎯 UX Улучшения
- **Прогресс-бар** процесса верификации (1/4, 2/4, 3/4, 4/4)
- **Визуальные индикаторы** статуса (✅ ❌ 🔄 ⏳)
- **Понятные сообщения** об ошибках
- **Возможность повторной попытки** через время
- **Анимированные эмодзи** для лучшего восприятия

## 🚀 Архитектура

### Компоненты системы:
1. **Telegram Bot** (`telegram-verification-bot.js`) - основная логика бота
2. **Website API** (`/api/telegram-verification`) - прием верификации от бота
3. **Verification Page** (`/auth/telegram-verified`) - обработка результата
4. **UI Components** - кнопки и формы для интеграции

### Поток данных:
```
Сайт → Telegram Bot → Верификация → Зашифрованный токен → Сайт → Регистрация
```

## 📋 Логика процесса

### Этап 1/4: Принятие условий
- Показ условий регистрации с прогресс-баром
- Проверка возраста аккаунта Telegram (≥7 дней)
- Валидация обязательных данных пользователя

### Этап 2/4: Вступление в группу
- Переход в группу @transformation_map_community
- Реальная проверка членства через Telegram API
- Валидация статуса участника

### Этап 3/4: Проверка безопасности  
- Математическая капча с 4 вариантами ответов
- **🔐 Система повторных попыток с прогрессивными сообщениями:**
  - 1-я ошибка: "Попробуйте еще раз"
  - 2-я ошибка: "Осталась 1 попытка"  
  - 3-я ошибка: "Блокировка на 5 минут"
- **🛡️ Защита от обхода:** проверка блокировки при показе и ответе
- **📊 Аналитика:** отслеживание всех попыток и блокировок

### Этап 4/4: Завершение
- Создание зашифрованного токена верификации
- Отправка данных на сайт через HTTPS API
- Перенаправление на форму регистрации

## 📡 API Endpoints

### Bot Webhook
- **URL**: `https://transformation-map.com/telegram-verification-webhook`
- **Method**: POST
- **Purpose**: Прием сообщений от Telegram

### Website API
- **URL**: `/api/telegram-verification`
- **Methods**: POST (получение верификации), GET (проверка статуса)
- **Purpose**: Обмен данными между ботом и сайтом

### Health Check
- **URL**: `http://localhost:9003/health`
- **Method**: GET
- **Response**: Статус бота, uptime, версия

### Statistics
- **URL**: `http://localhost:9003/stats`
- **Method**: GET  
- **Response**: Детальная аналитика конверсии

## 🔒 Безопасность

### Защитные механизмы:
1. **Шифрование токенов** - AES-256 для всех передаваемых данных
2. **Временные ограничения** - токены истекают через 30 минут
3. **Rate limiting** - ограничения на уровне пользователя и IP
4. **Проверка возраста аккаунта** - защита от свежих фейковых аккаунтов
5. **Капча** - математические примеры для защиты от ботов
6. **Nginx security headers** - дополнительная защита на уровне сервера

### Антиспам защита:
- Проверка реального членства в группе
- Лимит попыток регистрации
- Блокировка при превышении лимитов
- Проверка IP адресов на подозрительность

## 🚨 Устранение неполадок

### Бот не отвечает:
1. Проверить статус процесса: `pm2 status`
2. Посмотреть логи: `npm run bot:logs`
3. Проверить webhook: `npm run bot:status`
4. Перезапустить: `npm run bot:restart`

### Ошибки webhook:
1. Проверить Nginx: `nginx -t && systemctl status nginx`
2. Проверить доступность порта: `curl http://localhost:9003/health`
3. Проверить логи Nginx: `tail -f /var/log/nginx/error.log`

### Проблемы с верификацией:
1. Проверить API endpoint: `curl -X POST https://transformation-map.com/api/telegram-verification`
2. Проверить шифрование: сверить ключи в боте и на сайте
3. Проверить время жизни токенов
